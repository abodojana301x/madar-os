🌐 Madar Desktop — Stage 7 (Desktop Services v1.0)
🎯 الهدف

بناء طبقة خدمات خلفية (System Services) مسؤولة عن تشغيل النظام بشكل مستقر ومستمر، بحيث:

التطبيقات ما تعتمدش على بعضها مباشرة
كل حاجة لها “خدمة” مستقلة
النظام يبقى قابل للتوسع بدون كسر أي جزء
🧠 1. فكرة المرحلة (Core Concept)

بدل ما النظام يبقى:

Apps تتكلم مع بعض بشكل عشوائي

هنخليه:

Apps ↔ Services ↔ System Core

🧱 2. System Service Manager
🎯 مسؤول عن:
تشغيل الخدمات
إيقافها
مراقبتها
typedef struct {
    char* name;
    int running;

    void (*start)();
    void (*stop)();

} Service;

#define MAX_SERVICES 10

Service services[MAX_SERVICES];
int service_count = 0;

▶️ تشغيل خدمة
void start_service(Service* s) {
    if (!s->running) {
        s->start();
        s->running = 1;
    }
}

🔔 3. Notification Service
🎯 الهدف:

عرض إشعارات بسيطة على الشاشة

typedef struct {
    char message[128];
    int duration;
} Notification;

void notify(char* msg) {
    draw_notification(msg);
}

💡 مثال استخدام:
notify("Terminal opened");

📋 4. Clipboard Service
🎯 الهدف:

نقل النص بين التطبيقات

char clipboard[256];

void set_clipboard(char* text) {
    strcpy(clipboard, text);
}

char* get_clipboard() {
    return clipboard;
}

🧩 6. App Registry Service
🎯 الهدف:

تسجيل التطبيقات في النظام

typedef struct {
    char* name;
    char* exec_path;
    char* icon;
} App;

App apps[MAX_APPS];
int app_count = 0;

➕ إضافة تطبيق:
void register_app(char* name, char* path) {
    apps[app_count++] = (App){name, path, ""};
}

💾 7. Settings Persistence Service
🎯 الهدف:

حفظ إعدادات النظام

typedef struct {
    int theme;
    int language;
} Settings;

void save_settings(Settings s) {
    write_file("settings.cfg", &s);
}

🔄 8. System Event Bus (مهم جدًا)
🎯 الفكرة:

كل الخدمات تتواصل عبر Events مش مباشرة

typedef struct {
    char type[32];
    void* data;
} SystemEvent;

📡 إرسال حدث:
void emit_event(char* type) {
    dispatch(type);
}

🧠 9. Service Loop (قلب النظام)
while (system_running) {

}
