## 🖥️ **الطبقة السابعة: تطبيقات المستخدم والتكامل النهائي** (النسخة المفصّلة)

### 🎯 **الهدف العام**
تحويل النواة والخدمات والواجهة إلى **نظام تشغيل متكامل وجاهز للاستخدام** عبر:
- طرفية متعددة اللغات (Linux Style / Simple English / Arabic Native) متوافقة مع POSIX
- مدير ملفات رسومي، لوحة إعدادات، عميل شبكة خفيف، ومدير مهام تفاعلي
- تسلسل إقلاع سلس، شاشة تسجيل دخول، وعملية `init` تدير الخدمات تلقائياً
- اختبار ضغط واقعي (30 دقيقة) يضمن الاستقرار، استهلاك الموارد المنخفض، وخلو السجلات من الأخطاء

### 🚧 **الحدود الصارمة (Boundaries)**
| ✅ مسموح في هذه الطبقة | ❌ ممنوع تماماً |
|------------------------|----------------|
| تشغيل كـ تطبيقات User-Space تعتمد على Syscalls موحدة | أي منطق تطبيق يدخل النواة أو يعدّل الـ Kernel مباشرة |
| POSIX Compliance + دعم لغات متعددة في الـ Shell | مكتبات واجهة ثقيلة (GTK, Qt, Electron, WebKit) |
| HTTP/1.1 Basic + TCP Socket عبر Layer 4 | بروتوكولات تشفير معقدة (TLS 1.3) أو متصفحات كاملة |
| إدارة عمليات، موارد، وإقلاع تلقائي عبر `init` | حزم برامج (Package Manager) أو تحديثات OTA في v0.1 |
| واجهة خفيفة، استجابة <100ms، استهلاك ذاكرة <150MB | حظر الـ CPU في حلقات انتظار أو رسم متزامن ثقيل |

## 🧩 **المهام الدقيقة (6 مهام)**

### 🔹 المهمة 1: الطرفية متعددة اللغات (Multi-Language POSIX Shell)
#### 📁 الملفات المطلوبة
```
apps/
├── shell.c / shell.h           (حلقة REPL الرئيسية)
├── lang_detect.c               (كشف تلقائي: Linux / English / Arabic)
├── parsers/                    (linux_parser.c, english_parser.c, arabic_parser.c)
├── builtins.c / exec.c         (أوامر مدمجة + تنفيذ خارجي)
└── env_mgr.c                   (إدارة PATH, PS1, TERM, HOME)
```
#### 🔧 المواصفات التقنية
- **كشف اللغة:** يعتمد على كلمات مفتاحية أولية (`ls, cp` → Linux) / (`show, create` → English) / (`اعرض، انشئ` → Arabic)
- **محلل موحد (Unified AST):** كل الأنماط تُترجم لنفس الهيكل الداخلي `{cmd, flags, args, io_redirects}`
- **دعم POSIX:** Pipes `|`, Redirections `> < >>`, Background `&`, Globbing `*.txt`, Job Control `Ctrl+Z`
- **Prompt ديناميكي:** `user@madar:/path$ ` يدعم `PS1`، ألوان ANSI، و `TERM=xterm-256color`
- **الأمان:** Timeout للقراءة، عزل الـ Child Processes، منع Command Injection عبر فحص المسارات

#### ✅ معيار النجاح
```
🧪 Testing MultiLang Shell...
  Linux: ls -l /home | grep .txt > out.txt → OK
  English: show files in /home → Same output
  Arabic: اعرض ملفات /home → نفس النتيجة
  Pipes/Redirections: 100% pass, no blocking
✅ MultiLang Shell PASSED
```

#### 🆘 خطة الطوارئ
| المشكلة | الحل |
|---------|------|
| المحلل يعلق على رموز خاصة | فعّل `safe_escape()` يزيل أو يهمل الأحرف غير ASCII/UTF-8 الآمنة |
| أمر خارجي لا يستجيب | أضف `sys_kill_timeout(pid, 5s)` يقتل العملية تلقائياً |

## 🛡️ **البروتوكول الوقائي المدمج في الطبقة 7**

1. ✅ **User-Space Only:** كل التطبيقات تعمل بعيداً عن النواة. كراش أي تطبيق = موته فقط.
2. ✅ **Minimal Viable First:** نبدأ بالأوامر الأساسية، ملفات بسيطة، HTTP نصي، ومدير مهام خفيف. التوسع لاحقاً.
3. ✅ **Testing Checklist إلزامي:** كل مهمة فيها `test_*()` تطبع `PASSED/FAILED` على Serial. لا انتقال بدون نجاح.
4. ✅ **Clear Syscall Boundaries:** لا اتصال مباشر مع الهاردوير أو النواة إلا عبر واجهات Layer 2/4/6 الموثقة.
5. ✅ **Fallback & Safe Mode:** كل تطبيق يدعم وضع نصي أو آمن لو الواجهة فشلت. الإقلاع يدعم `Safe Boot`.
6. ✅ **Documentation & Logging:** كل ملف يبدأ بـ Header يوضح الطبقة، الوظيفة، والـ dependencies. كل خطأ يُسجل مع Stack Trace مبسط.
