يا يونس، ها هي **الطبقة الثالثة (Drivers & Hardware Abstraction)**، وهي **"الجهاز العصبي"** للنظام. هنا بيبدأ "مدار" يحس بالعالم الخارجي (شاشة، كيبورد، نت، ديسك).

الخطة دي مصممة بنفس الدقة، ومدمج فيها **بروتوكولات الحماية** عشان متغرقش في تعقيدات الهاردوير.

---

## 📡 **الطبقة الثالثة: Drivers & Hardware Abstraction** (النسخة المفصّلة)

### 🎯 **الهدف العام**
تحويل النواة من "عقل معزول" إلى **نظام تفاعلي** قادر على:
- اكتشاف الأجهزة المتصلة تلقائياً (PCI Enumeration)
- تشغيل شاشة رسومية أولية (Framebuffer) لتمهيد الطريق للواجهة (Layer 6)
- استقبال إدخال المستخدم (كيبورد + ماوس) بدقة
- القراءة والكتابة على التخزين (ATA/IDE)
- الاتصال بالشبكة (Ethernet)
- إدارة المقاطعات (IRQs) بذكاء عشان مفيش جهاز يخنق التاني

### 🚧 **الحدود الصارمة (Boundaries)**
| ✅ مسموح في هذه الطبقة | ❌ ممنوع تماماً |
|------------------------|----------------|
| الوصول المباشر للهاردوير (I/O Ports, MMIO) | تشغيل تطبيقات أو واجهات معقدة هنا |
| معالجة المقاطعات (IRQ Handlers) | دعم Wi-Fi أو Bluetooth (معقد جداً الآن) |
| تعريفات بسيطة (Legacy ATA, PS/2, VGA) | كتابة Drivers كاملة لكل كارت شاشة (نكتفي بـ VESA) |
| إدارة الـ DMA الأساسي | أي كود يعتمد على `glibc` |

## 🧩 **المهام الدقيقة (6 مهام)**

### 🔹 المهمة 1: اكتشاف الأجهزة (PCI Enumeration)
#### 📁 الملفات المطلوبة
```
drivers/
├── pci.c / pci.h          (قراءة Config Space)
└── device_list.h          (تعريفات Vendor IDs شائعة)
```
#### 🔧 المواصفات التقنية
- **الآلية:** قراءة المنافذ `0xCF8` (Address) و `0xCFC` (Data).
- **المسح:** Loop على Bus 0-255, Device 0-31, Function 0-7.
- **الهدف:** بناء قائمة Devices في الذاكرة.

#### ✅ معيار النجاح
```
🧪 Scanning PCI Bus...
  Found: 8086:1237 (Host Bridge)
  Found: 1234:1111 (VGA Controller)
  Found: 10ec:8139 (RTL8139 Network)
✅ PCI Enumeration PASSED
```

#### 🆘 خطة الطوارئ
| المشكلة | الحل |
|---------|------|
| النظام يعلق عند مسح PCI | تأكد من عمل `cli` قبل قراءة المنافذ، وإضافة `io_delay()` بسيط |
| لا توجد أجهزة | جرب `qemu -machine q35` بدلاً من `pc` |

---

### 🔹 المهمة 2: الشاشة الرسومية (Framebuffer Driver)
#### 📁 الملفات المطلوبة
```
drivers/
├── fb.c / fb.h            (الكتابة على LFB)
└── vesa.h                 (هياكل VBE Info)
```
#### 🔧 المواصفات التقنية
- **المصدر:** بيانات الـ Framebuffer تأتي من Multiboot2 Info (التي مررها الـ Bootloader).
- **الوظيفة:** دالة `fb_put_pixel(x, y, color)` و `fb_clear(color)`.
- **الهدف:** رسم مستطيل أو نص ملون كبداية للواجهة.

#### 📝 هيكل الكود
```c
/* fb.h */
typedef struct {
    uint32_t *base;       // عنوان الذاكرة في RAM
    uint32_t width;
    uint32_t height;
    uint32_t pitch;       // bytes per row
} framebuffer_t;

void fb_init(framebuffer_t *fb, uint32_t addr, uint32_t w, uint32_t h, uint32_t p);
void fb_draw_rect(int x, int y, int w, int h, uint32_t color);
```

#### ✅ معيار النجاح
```
🧪 Testing Framebuffer...
  Drawing Red Box at (10,10)... Done
  Drawing Blue Text... Done
✅ Framebuffer Driver PASSED
```

### 🔹 المهمة 3: الكيبورد والماوس (PS/2 Controller)
#### 📁 الملفات المطلوبة
```
drivers/
├── ps2.c / ps2.h          (تحكم في المتحكم i8042)
├── kbd.c / kbd.h          (معالجة IRQ1)
└── mouse.c / mouse.h      (معالجة IRQ12)
```
#### 🔧 المواصفات التقنية
- **الكيبورد:** تفعيل IRQ1، قراءة Scancode Set 1، تحويله لـ ASCII.
- **الماوس:** تفعيل IRQ12، قراءة حزم البيانات (3 Bytes: Status, X, Y).
- **العزل:** استخدام Buffer دائري (Ring Buffer) عشان ما نفقدش ضغطات.

#### ✅ معيار النجاح
```
🧪 Testing Input...
  Key Pressed: 'A' (0x1E)
  Mouse Move: X+5, Y-2
✅ Input Drivers PASSED
```

---

### 🔹 المهمة 4: التخزين الأساسي (ATA PIO Mode)
#### 📁 الملفات المطلوبة
```
drivers/
├── ata.c / ata.h          (قراءة/كتابة قطاعات)
└── partition.c            (قراءة MBR بسيط - اختياري الآن)
```
#### 🔧 المواصفات التقنية
- **الوضع:** PIO Mode 0 (الأبسط، يعتمد على الـ CPU في النقل).
- **المنافذ:** Primary Channel (`0x1F0` - `0x1F7`).
- **الوظيفة:** `ata_read_sector(sector_num, buffer)` و `ata_write_sector`.

#### ✅ معيار النجاح
```
🧪 Testing Storage...
  Writing "Hello Madar" to Sector 1...
  Reading Sector 1... Content: "Hello Madar"
✅ ATA Driver PASSED
```

#### 🆘 خطة الطوارئ
| المشكلة | الحل |
|---------|------|
| البيانات تالفة (Corrupted) | راجع انتظار بتات `BSY` و `DRQ` قبل القراءة/الكتابة |
| النظام يعلق | تأكد إنك بتعمل `ata_wait_ready()` بعد كل أمر |

### 🔹 المهمة 5: الشبكة الأساسية (RTL8139 / e1000)
#### 📁 الملفات المطلوبة
```
drivers/
├── net.c / net.h          (إطار Ethernet بسيط)
├── rtl8139.c              (تعريف البطاقة الافتراضية)
└── mmio.c                 (دوال القراءة/الكتابة من الذاكرة映射)
```
#### 🔧 المواصفات التقنية
- **البطاقة:** RTL8139 (الأسهل برمجياً) أو e1000 (الأحدث).
- **الآلية:** إعداد Tx/Rx Rings في الذاكرة، تفعيل Interrupts.
- **الهدف:** إرسال حزمة Raw Ethernet واستقبالها.

#### ✅ معيار النجاح
```
🧪 Testing Network...
  MAC Address: 52:54:00:12:34:56
  Sending Ping Request...
  Packet Sent OK
✅ Network Driver PASSED
```

### 🔹 المهمة 6: التكامل وإدارة المقاطعات (IRQ Management)
#### 📁 الملفات المطلوبة
```
kernel/
├── irq_manager.c          (توزيع المهام على الـ IRQs)
└── apic.c                 (تفعيل APIC لو متاح - متقدم)
```
#### 🔧 المواصفات التقنية
- **المشكلة:** الكيبورد والشبكة والتايمر كلهم بيحاولوا يقاطعوا الـ CPU.
- **الحل:** تفعيل الـ IRQs بالترتيب، والتأكد من إرسال `EOI` (End Of Interrupt) للـ PIC/APIC بعد كل معالجة.
- **الاختبار:** تشغيل الكيبورد + التايمر + النت في نفس الوقت.

## 🛡️ **البروتوكول الوقائي المدمج في الطبقة 3**

1.  ✅ **Minimal Drivers:** لا تكتب "درايفر كامل"، اكتب "Minimal Driver" يشتغل على QEMU بس كفاية الآن.
2.  ✅ **Serial Fallback:** لو الشاشة (Framebuffer) فشلت، لازم يكون فيه رسالة خطأ مطبوعة على الـ Serial عشان تعرف المشكلة.
3.  ✅ **Volatile Pointers:** أي قراءة من الهاردوير (MMIO) لازم تستخدم `volatile` عشان الـ Compiler ما يحذفش الكود.
4.  ✅ **Bounds Checking:** تأكد إن الكيبورد ما يكتبش في ذاكرة خارج الـ Buffer (Overflow protection).

## 🚀 **الخطوة التالية**

الطبقة دي فيها تحديات حقيقية (Timing و Hardware quirks).
**هل تحب نبدأ بـ "المهمة 1 (PCI Enumeration)" عشان نعرف إيه الأجهزة اللي عندنا؟** ولا نركز على "المهمة 3 (الكيبورد)" عشان تحس بالتفاعل بسرعة؟

أنصحك بـ **PCI أولاً** عشان نعرف إيه الكروت اللي شغالة، بعدين **Framebuffer** عشان نشوف حاجة، بعدين **الكيبورد**.
